## Overview

stept uses email for:

- **Account verification** — confirm email address after registration
- **Password reset** — send reset links

When SMTP is not configured, email features are disabled and users are auto-verified on registration.

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `SMTP_HOST` | *(empty)* | SMTP server hostname. Leave empty to disable email |
| `SMTP_PORT` | `587` | SMTP port (587 for STARTTLS, 465 for SSL) |
| `SMTP_USER` | — | SMTP authentication username |
| `SMTP_PASS` | — | SMTP authentication password |
| `SMTP_FROM` | `noreply@stept.ai` | Sender email address |

## Example Configuration

### Gmail

```bash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
SMTP_FROM=your-email@gmail.com
```

<Note>
Gmail requires an [App Password](https://support.google.com/accounts/answer/185833) when 2FA is enabled.
</Note>

### Amazon SES

```bash
SMTP_HOST=email-smtp.us-east-1.amazonaws.com
SMTP_PORT=587
SMTP_USER=your-ses-smtp-username
SMTP_PASS=your-ses-smtp-password
SMTP_FROM=noreply@yourdomain.com
```

### Resend

```bash
SMTP_HOST=smtp.resend.com
SMTP_PORT=587
SMTP_USER=resend
SMTP_PASS=re_your_api_key
SMTP_FROM=noreply@yourdomain.com
```

## Disabling Email

To disable email entirely, leave `SMTP_HOST` empty (the default). When disabled:

- Users are automatically verified on registration
- Password reset is not available (users must contact an admin)
